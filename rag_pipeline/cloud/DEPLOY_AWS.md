# Deploying the Biosensor RAG assistant on AWS

The AWS twin of `DEPLOY_GCP.md`. Same pluggable pipeline — only the `*_backend`
modules change. Everything runs locally for free; this is the path to ship it on
AWS. The pgvector path even runs **fully locally on Docker** (no AWS account),
because the same code targets a local Postgres or Amazon RDS/Aurora.

| Concern            | Local (default, free)        | AWS (cloud-native)                              |
|--------------------|------------------------------|-------------------------------------------------|
| Embeddings         | TF-IDF/LSA or MiniLM (local) | Amazon Bedrock — Titan Text Embeddings v2       |
| Vector search      | FAISS / numpy (in-process)   | **pgvector** on RDS/Aurora **or** Amazon OpenSearch Serverless k-NN |
| Generation (LLM)   | extractive / Ollama          | Claude on Amazon Bedrock                         |
| Data warehouse     | local CSV/JSON               | Amazon Athena over S3 **or** Amazon Redshift     |
| Serving            | Flask (`app.py`)             | App Runner / ECS Fargate (the same `Dockerfile`) |
| CI/CD              | —                            | CodeBuild / GitHub Actions → ECR → App Runner    |

Backend modules: `cloud/aws_bedrock.py` (embeddings + LLM), `cloud/aws_pgvector.py`
(open-source vector DB), `cloud/aws_opensearch.py` (managed k-NN).

---

## 0. One-time setup
```bash
pip install -r requirements.txt -r requirements-aws.txt
aws configure                      # access key / secret / region (e.g. us-east-1)
# In the Bedrock console (Model access), enable: Titan Text Embeddings V2 + a Claude model.
```

## A. pgvector — open-source vector DB, vector search in SQL  (runs locally, $0)
```bash
# local Postgres with the pgvector extension, via Docker — no AWS needed:
docker run -d --name ragpg -p 5432:5432 \
  -e POSTGRES_USER=rag -e POSTGRES_PASSWORD=rag -e POSTGRES_DB=rag pgvector/pgvector:pg16

export RAG_PG_DSN="postgresql://rag:rag@localhost:5432/rag"
python cloud/aws_pgvector.py "wearable sensors that measure cortisol in sweat"
#   -> embeds the 639-abstract corpus (TF-IDF, $0), loads into pgvector,
#      and runs k-NN entirely in SQL:  ORDER BY embedding <=> :q LIMIT k
```
**Prod:** point `RAG_PG_DSN` at an Amazon RDS / Aurora PostgreSQL endpoint (which
ships the `vector` extension). Same code, managed database.

## B. Amazon Bedrock embeddings + Claude  (real AWS services)
```bash
# Titan embeddings into the SAME pgvector table (1024-dim instead of 256):
RAG_EMBED_BACKEND=bedrock python cloud/aws_pgvector.py "cortisol in sweat"

# A grounded, PMID-cited answer generated by Claude on Bedrock:
python build_index.py --no-fact-cards
RAG_GEN_BACKEND=bedrock python ask.py "Which wearable sensors measure cortisol in sweat?"
```

## C. Amazon OpenSearch Serverless — managed k-NN (cloud-native vector search)
```bash
# create a Serverless collection (type=VECTORSEARCH) + a data-access policy for your IAM principal, then:
export RAG_OPENSEARCH_HOST=abc123.us-east-1.aoss.amazonaws.com   # collection endpoint, no https://
python -c "from cloud.aws_opensearch import create_index; print(create_index(1024))"
# index your embedded chunks with index_documents(...) and query with knn_search(...)
```

## Notes
- **Open-source vs. managed vector store:** pgvector (Postgres extension, fully
  open source) for dev and small/medium corpora; Amazon OpenSearch Serverless
  k-NN when you want a managed ANN service at scale — same retrieval interface.
- **Data warehousing + SQL:** the structured fact-card table lands in S3/Athena or
  Redshift; analysts query it with plain SQL (the `biomarker_rollup_sql` rollup
  works there too). This is the BigQuery analog from the GCP path.
- **Data privacy (public health):** PMIDs/abstracts are public; over PII the AWS
  pattern adds **Amazon Comprehend (PII detection)** or **Amazon Macie** for
  de-identification before indexing — the analog of Google DLP.
- **Portability:** the same `embeddings.py` / `vector_store.py` / `generator.py`
  interfaces back the GCP path; AWS just swaps Bedrock + pgvector/OpenSearch for
  Vertex + BigQuery/Vector Search. One env var moves between clouds.
- **Cost control:** Titan embeds the ~640-row corpus for fractions of a cent and
  is computed once; generation is metered per request (every answer returns a
  `usage` block with `est_cost_usd`).
